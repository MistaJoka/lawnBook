# API and App Actions

## Purpose

Define how the frontend talks to backend logic, database operations, and external services.

## Endpoint / Action Template

```md
## Action Name

### Purpose

### Input

### Output

### Errors

### Auth Required

### Offline Behavior
```

## Questions

### Beginner
- What data does the app create?
- What data does the app read?
- What data does the app update?
- What data does the app delete?
- What external data does the app fetch?

### Advanced
- What requests need authentication?
- What actions need rate limiting?
- What actions need validation?
- What errors should be user-friendly?
- What API calls should be retried?
- What API calls should never be retried automatically?
