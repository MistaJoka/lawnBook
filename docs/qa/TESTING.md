# Testing Plan

## Test Philosophy

The MVP is not done until the core user flow works repeatedly without confusion or data loss.

## Manual Test Checklist

- [ ] App starts locally
- [ ] Main screen loads
- [ ] User can create core record
- [ ] User can view core record
- [ ] User can edit core record
- [ ] User can delete/archive core record
- [ ] Empty state works
- [ ] Error state works
- [ ] Mobile viewport works
- [ ] Refresh does not break state

## Questions

### Beginner
- What is the happy path?
- What should be tested manually first?
- What should never break?
- What data should be tested with?

### Advanced
- What unit tests are needed?
- What integration tests are needed?
- What end-to-end tests are needed?
- What regression tests are needed before each release?
- What should be mocked?
